# Blender 骨骼動畫錯誤知識譜

## 2026-03-05 17:40 發現

### 錯誤現象
- 注入的 Walk 動畫導致模型嚴重變形
- 四肢拉伸、關節扭曲
- 完全不像行走動作

### 根本原因
**Blender 導出破壞了骨骼的預設姿勢（Rest Pose）**

#### 具體問題
| 項目 | 原始模型 | Blender 導出 |
|------|----------|--------------|
| Hip Translation | `[0, 0, 111.6]` | `[0, 0, 0]` ❌ |
| Hip Rotation | `[0.707, 0, 0, 0.707]` | `[0, 0, 0, 1]` ❌ |
| IBM | 正確的蒙皮矩陣 | 錯誤的蒙皮矩陣 ❌ |

#### 連鎖效應
1. Blender 導出時，把所有骨骼的 translation/rotation 都重置了
2. 基於錯誤的預設姿勢製作的動畫
3. 注入到原始模型時，動畫數據和原始預設姿勢不匹配
4. 導致嚴重變形

### 錯誤的修復嘗試

#### 嘗試 1：修復骨骼 translation
- ✅ 修復了 translation
- ❌ rotation 和 IBM 仍然錯誤
- 結果：模型變形

#### 嘗試 2：修復 translation + rotation + IBM
- ✅ 所有數據都修復了
- ❌ 但動畫是基於錯誤預設姿勢計算的
- 結果：動畫導致變形

#### 嘗試 3：注入動畫到原始模型
- ✅ 保留原始結構
- ❌ 動畫數據仍然基於錯誤預設
- 結果：動畫導致嚴重變形

### 正確的解決方案

#### 方案 A：在 Blender 裡設置正確的預設姿勢
1. 導入原始模型
2. **保留骨骼的預設 translation/rotation**（不要讓 Blender 重置）
3. 基於這個正確的預設姿勢製作動畫
4. 導出時只導出動畫數據

#### 方案 B：使用 Mixamo（推薦）
1. 上傳模型到 Mixamo
2. 選擇行走動畫
3. 自動重定向骨骼
4. 下載帶動畫的模型

#### 方案 C：程序化動畫
1. 不依賴 Blender
2. 用 Three.js 直接生成動畫數據
3. 參考原始模型自帶的動畫結構

### 關鍵教訓

1. **glTF 的預設姿勢至關重要**
   - 所有動畫都基於預設姿勢計算
   - 預設姿勢錯誤 = 動畫錯誤

2. **Blender 5.0.1 的 glTF 導出器有嚴重 Bug**
   - 會破壞骨骼 translation
   - 會破壞骨骼 rotation
   - 會破壞 inverseBindMatrix

3. **不能只修復部分數據**
   - Translation、Rotation、IBM 必須全部正確
   - 動畫數據必須基於正確的預設姿勢

4. **測試流程**
   - 先確認靜態模型顯示正確
   - 再測試動畫
   - 如果靜態模型變形，動畫一定也會變形
